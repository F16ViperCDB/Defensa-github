01

¿Qué es y por qué usarlo?

Sistema de control de versiones (VCS)

Herramienta que registra cada cambio realizado en el código a lo largo del tiempo, guardando "instantáneas" (commits) del proyecto. Permite volver a cualquier estado anterior, ver quién cambió qué y trabajar en paralelo sin sobrescribir el trabajo de otros. Git es un VCS distribuido: cada desarrollador tiene una copia completa del historial.

✓
Historial completoRecupera versiones previas y compara cambios.
✓
Trabajo en equipoVarias personas colaboran sin conflictos.
✓
Ramas (branches)Prueba ideas sin afectar la versión estable.
✓
Respaldo distribuidoEl código vive en cada máquina y en la nube.
✓
TrazabilidadCada cambio queda firmado por su autor.
✓
Integración continuaBase para automatizar pruebas y despliegues.

Instalación del cliente Git

Elige tu sistema operativo. Al terminar, verifica la instalación con git --version.

⊞ Windows
  1. Descarga Git for Windows desde git-scm.com.
  2. Ejecuta el instalador (.exe) y acepta las opciones por defecto.
  3. Se incluye Git Bash como terminal.
$ git --version
⌘ macOS
  1. Instala Homebrew si no lo tienes.
  2. Ejecuta el comando en la Terminal.
  3. También viene con las Xcode Command Line Tools.
$ brew install git
🐧 Linux
  1. Abre la terminal.
  2. Usa el gestor de paquetes de tu distro.
  3. Confirma la versión instalada.
$ sudo apt install git
03

Comandos más comunes

Configurar & iniciar

git configDefine nombre y correo del usuario.
git initCrea un repositorio nuevo en la carpeta.
git cloneCopia un repositorio remoto a tu equipo.

Trabajo diario

git statusMuestra archivos modificados o pendientes.
git addPrepara cambios para el próximo commit.
git commit -mGuarda una instantánea con un mensaje.
git logLista el historial de commits.

Ramas

git branchLista o crea ramas de trabajo.
git checkoutCambia de rama o restaura archivos.
git mergeUne los cambios de una rama en otra.

Sincronizar con GitHub

git remote addConecta tu repo local con GitHub.
git pushEnvía tus commits al repositorio remoto.
git pullTrae los cambios del remoto a tu equipo.
git fetchDescarga cambios sin fusionarlos aún.
Flujo de trabajo básico
04

Publicar un sitio web con GitHub Pages

GitHub puede funcionar como servidor gratuito para sitios estáticos (HTML, CSS y JavaScript) mediante GitHub Pages. En cinco pasos tu página estará en línea.

Crea el repo

Nuevo repositorio público en tu cuenta de GitHub.

Sube tu sitio

Coloca index.html y haz push del código.

Abre Settings

Entra a la pestaña Settings › Pages del repo.

Elige la rama

Selecciona main como fuente y guarda.
